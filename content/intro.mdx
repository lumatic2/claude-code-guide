# Claude Code 입문 소개

> **당신은 코드를 쓸 필요가 없어집니다. 대신 무엇을 만들지만 말하세요.**

Claude Code는 단순한 AI 어시스턴트가 아닙니다. **당신의 지시**를 받으면 파일을 직접 열고, 코드를 수정하고, 명령을 실행하는 **AI 에이전트**입니다. 마치 경험 많은 개발자가 옆에서 당신의 지시를 따르듯이 말이죠.

## 챗봇과 에이전트의 차이

### 일반 AI 챗봇 vs Claude Code

**일반 AI 챗봇(Copilot, ChatGPT 등)**
- "이 버그를 어떻게 고칠까요?" → 해결 방법을 제안만 함
- 당신이 직접 파일을 열고, 코드를 수정하고, 테스트를 실행
- 일종의 "조언가"

**Claude Code (AI 에이전트)**
- "이 버그를 고쳐줘" → 파일을 열고 직접 수정하고 테스트까지 실행
- 당신은 결과만 확인
- 일종의 "실행가"

<Callout type="tip">
에이전트라는 이름처럼, Claude Code는 마치 자동으로 일을 처리하는 로봇처럼 작동합니다.
</Callout>

## 무엇이 가능한가?

### 예시 1: 새 기능 구현

```bash
당신: "React 컴포넌트를 만들어줘. 버튼을 클릭하면 숫자가 증가하는 카운터"

Claude Code의 행동:
1. src/Counter.jsx 파일 생성
2. 카운터 로직 + UI 작성
3. 테스트 실행해서 동작 확인
4. 결과 보고
```

### 예시 2: 버그 수정

```bash
당신: "로그인이 안 돼. auth.js에서 뭐가 잘못됐는지 찾아서 고쳐줘"

Claude Code의 행동:
1. auth.js 파일 읽기
2. 에러 원인 분석
3. 코드 수정
4. 테스트 실행
5. "이 부분이 문제였고 이렇게 고쳤습니다" 설명
```

### 예시 3: 성능 최적화

```bash
당신: "웹사이트가 느려. 어디가 문제고 어떻게 개선할지 분석해줘"

Claude Code의 행동:
1. 모든 파일 분석
2. 느린 부분 찾기
3. 최적화 코드 작성
4. 개선 결과 측정
```

## 다른 도구와 비교

| 도구 | 특징 | 한계 |
|---|---|---|
| **GitHub Copilot** | IDE 자동완성, 코드 제안 | 파일 쓰기만 가능, 터미널 실행 X |
| **Cursor** | IDE 통합, 코드 편집 | 한 번에 여러 파일 수정 어려움 |
| **Claude Code** | 터미널 기반, 자동 파일 관리, 명령 실행 | 한 번에 한 세션만 가능 |

## 이 가이드의 학습 로드맵

이 웹사이트는 다음 순서대로 학습하도록 설계되었습니다:

1. **설치 & 첫 실행** — 터미널 설정, Claude Code 설치
2. **핵심 사용법** — 명령어 5가지, 요청 작성법
3. **실전 실습** — 실제 프로젝트 3개 따라하기
4. **치트시트** — 자주 쓰는 명령어 한 장 정리
5. **고급 활용** — 복잡한 작업, 팀 협업
6. **트러블슈팅** — 자주 나오는 문제와 해결법

각 섹션은 이전 내용을 기반으로 하므로 순서대로 읽는 것을 추천합니다.
