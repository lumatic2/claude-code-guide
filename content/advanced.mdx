# 고급 설정

## CLAUDE.md

`CLAUDE.md`는 에이전트 동작 원칙을 고정하는 핵심 설정 파일입니다.

- 전역 `CLAUDE.md`: 사용자 홈 경로에 두고, 여러 프로젝트에 공통 적용합니다.
- 프로젝트 `CLAUDE.md`: 저장소 루트에 두고, 팀 규칙과 작업 범위를 구체화합니다.
- 충돌 시에는 더 구체적인 프로젝트 규칙을 우선으로 관리하는 것이 안전합니다.

컨텍스트가 길어질수록 지시 우선순위가 흐려질 수 있으므로, 작업 전 `목표/범위/완료 기준`을 다시 명시해 문맥을 재정렬하세요.

<Callout type="tip" title="팁">
전역에는 공통 원칙만 두고, 프로젝트에는 파일 경로/금지 영역/테스트 명령처럼 실행 가능한 규칙만 넣으면 충돌을 줄일 수 있습니다.
</Callout>

## Hooks

Hooks는 특정 시점에 자동으로 검증이나 후처리를 붙일 수 있는 실행 지점입니다.

- `SessionStart`: 세션 시작 시 환경 점검, 브랜치 확인, TODO 로드
- `PreToolUse`: 명령 실행 전 위험 커맨드 차단, 인자 검증
- `PostToolUse`: 결과 로그 수집, 실패 패턴 감지, 후속 알림
- `Notification`: 외부 알림 전송 (Slack, 데스크톱 알림 등)
- `Stop`: 세션 종료 전 요약 저장, 임시 산출물 정리

자동화는 생산성을 높이지만, 차단형 Hook은 오탐 시 작업을 멈출 수 있으니 작은 규칙부터 점진적으로 적용하세요.

<Callout type="tip" title="팁">
`PreToolUse`에는 보안성 규칙(예: `git reset --hard` 차단)을, `PostToolUse`에는 관측성 규칙(로그/메트릭)을 배치하면 운영이 단순해집니다.
</Callout>

## Skills

Skills는 재사용 가능한 작업 절차를 묶어 슬래시 명령으로 빠르게 실행하는 방식입니다.

- 팀에서 반복되는 작업을 Skill로 표준화하면 품질 편차를 줄일 수 있습니다.
- 명령어는 짧고 의미가 명확해야 합니다.

예시:

```bash
/review
/ship
```

- `/review`: 코드 리뷰 체크리스트(버그, 보안, 테스트 누락) 기반 점검
- `/ship`: 릴리스 준비(빌드, 테스트, 변경 요약, 배포 체크) 자동 실행

<Callout type="tip" title="팁">
Skill은 "입력 형식 + 실행 단계 + 완료 조건" 3요소를 고정하면 팀원 간 결과가 일관됩니다.
</Callout>

## Sub-agents

Sub-agent는 메인 에이전트가 작업을 분할 위임할 때 쓰는 독립 실행 단위입니다.

- 격리 컨텍스트: 부모 문맥을 필요한 만큼만 전달해 간섭을 줄입니다.
- 내장 타입: `explorer`, `worker`, `reviewer` 등 역할별 특화 에이전트를 선택합니다.
- `Agent()` 위임: 탐색/구현/검증을 병렬로 분리해 처리 시간을 단축합니다.

Sub-agent를 늘릴수록 조율 비용이 커지므로, 의존성이 낮은 작업부터 분리하는 것이 효율적입니다.

<Callout type="tip" title="팁">
먼저 "파일 소유 범위"를 나눈 뒤 위임하면 충돌을 크게 줄일 수 있습니다.
</Callout>

## Permissions

Permissions는 도구 실행 권한을 단계적으로 통제해 오작동과 권한 오남용을 방지합니다.

- 1단계: 기본 샌드박스 실행 (읽기/제한된 쓰기)
- 2단계: 승인 기반 상승 권한 (사용자 확인 후 실행)
- 3단계: 완전 권한 모드 (신뢰된 환경에서만 사용)

`allowlist`를 사용하면 허용된 명령/경로만 통과시켜 위험 명령을 원천 차단할 수 있습니다. 세션 중 권한 상태는 `/permissions`로 즉시 확인/조정하세요.

<Callout type="tip" title="팁">
기본 정책은 최소 권한으로 두고, 필요한 순간에만 임시 상승하는 방식이 가장 안전합니다.
</Callout>
