# 치트시트

## 슬래시 명령어

| 명령어 | 기능 |
|--------|------|
| `/help` | 도움말 |
| `/clear` | 컨텍스트 초기화 |
| `/compact` | 컨텍스트 압축 (긴 대화 시 유용) |
| `/model` | 모델 변경 |
| `/cost` | 세션 비용 확인 |
| `/exit` | 종료 |

## 유용한 프롬프트 패턴

| 상황 | 패턴 |
|------|------|
| 파일 수정 | "X 파일에서 Y를 Z로 바꿔줘" |
| 기능 추가 | "A 기능을 B에 추가해줘" |
| 디버깅 | "이 에러 고쳐줘: [에러 메시지]" |
| 코드 리뷰 | "이 코드 검토해줘" |
| 설명 요청 | "X 파일 어떻게 동작하는지 설명해줘" |
| 테스트 작성 | "X 함수에 대한 유닛 테스트 작성해줘" |

## 워크플로우 팁

<Callout type="tip" title="커밋 먼저">
큰 변경 전에 `git commit`으로 체크포인트를 만들어두세요.
</Callout>

<Callout type="tip" title="범위 명시">
"전체 프로젝트" 대신 특정 파일/함수를 지정하면 더 정확합니다.
</Callout>

<Callout type="tip" title="반복 세션">
작업이 많으면 `/clear`로 컨텍스트를 리셋하고 새 작업을 시작하세요.
</Callout>

## 단축키

| 키 | 동작 |
|----|------|
| `Ctrl+C` | 현재 작업 중단 |
| `↑` / `↓` | 히스토리 탐색 |
| `Ctrl+L` | 화면 지우기 |
