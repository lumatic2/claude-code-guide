# 실전 워크플로우

Claude Code로 실제 개발 작업을 효율적으로 진행하는 4가지 워크플로우를 소개합니다.

## 버그 수정 세션

오류가 발생했을 때 증상부터 설명하세요. Claude가 원인을 파악하고 수정을 제안합니다.

**동작:**
1. 오류 메시지와 상황 설명
2. Claude가 코드 분석 → 원인 파악 → 수정 제안
3. 수정 적용 후 검증

**실제 대화 예시:**

```
You: login() 함수를 호출하면 "Cannot read property 'email' of undefined" 오류가 납니다. users 배열이 비어있는 건가요?

Claude: 코드를 확인했습니다. auth.js의 15번 줄에서 user 객체가 null 체크 없이 바로 접근하고 있네요.

수정:
- 15번 줄: if (user?.email) 로 변경
- 또는 login 함수 시작 부분에서 if (!user) return null 추가

You: 적용해줘

Claude: auth.js 수정 완료했습니다. 이제 null 값도 안전하게 처리됩니다.
테스트해보세요!
```

## 새 기능 구현 세션

먼저 계획을 세운 후 단계별로 구현하세요. 큰 작업은 작은 단위로 나누어 진행합니다.

**동작:**
1. `/plan` 으로 설계 및 계획 수립
2. Claude가 단계별 구현 제시
3. 각 단계마다 파일 수정 + 검증

**실제 대화 예시:**

```
You: 사용자 프로필 페이지를 만들어야 합니다. 사진 업로드, 이름·이메일 수정 기능이 필요합니다.

Claude: /plan으로 설계부터 시작할까요?

You: /plan

Claude: 
## 구현 계획
1. API 라우트 생성 (POST /api/user/update)
2. 프로필 폼 컴포넌트 작성
3. 이미지 업로드 로직 추가
4. 유효성 검사 및 에러 처리

You: 1번부터 시작해줘

Claude: app/api/user/update/route.ts 파일을 만들고... [구현 완료]
```

## 코드 리뷰 세션

파일을 @ 기호로 참조하면 Claude가 코드를 분석하고 개선점을 제안합니다.

**동작:**
1. `@파일경로` 로 파일 전달
2. Claude가 코드 분석
3. 버그, 성능, 스타일 개선점 제시
4. 수정 검토

**예시:**

```
You: @src/auth.js 이 파일을 리뷰해줄 수 있나요?

Claude: auth.js를 분석했습니다.

개선점:
1. 5번 줄: 에러 처리 없음 → try-catch 추가 권장
2. 12번 줄: 비동기 처리 missing await
3. 성능: 매번 전체 사용자 배열 검색 → 인덱스 추가

수정 파일을 제시해줄까요?

You: 네, 적용해줘
```

## 안전하게 작업하기

큰 변경 전에는 항상 체크포인트를 만드세요.

| 동작 | 방법 |
|------|------|
| 직전 상태로 복구 | `Esc × 2` 또는 `/rewind` |
| 현재 상태 저장 | `!git commit -m "메시지"` |
| 이전 버전 확인 | `git log` |

<Callout type="warn">
부수 효과(외부 API 호출, DB 변경)는 `/rewind`로 되돌릴 수 없습니다. 큰 작업 전에는 `!git commit`으로 체크포인트를 만드세요.
</Callout>

## 반복 작업 자동화 (CLAUDE.md)

프로젝트별로 반복되는 지시사항을 `CLAUDE.md`에 등록하면 Claude가 매번 참고합니다.

**예시 (CLAUDE.md):**

```markdown
# My Project Guide

- 모든 React 컴포넌트는 함수형으로 작성
- TypeScript 필수 (any 금지)
- 테스트: Jest + React Testing Library
- 커밋 메시지: "기능: 설명" 형식
- 이슈 번호는 커밋에 포함 (#123)

## 폴더 구조
- src/components/ → UI 컴포넌트
- src/hooks/ → Custom hooks
- src/api/ → API 호출 로직
```

파일을 저장하면 Claude Code가 자동으로 읽고 모든 작업에 반영합니다.

**장점:**
- 스타일 가이드 일관성 유지
- 반복 설명 불필요
- 팀 규칙 자동 적용
