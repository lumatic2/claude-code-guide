# 실전 워크플로우

말 한마디로 실제 결과물을 만드는 4가지 워크플로우입니다. 순서대로 따라해보세요.

## 블로그 · 문서 작성

글을 쓰거나 문서를 정리할 때 Claude Code가 초안부터 다듬기까지 도와줍니다.

<StepCard step={1} title="주제와 방향 전달하기">
어떤 글인지, 어떤 느낌으로 쓸지 알려주세요. 자료가 있으면 같이 전달하면 더 좋습니다.
</StepCard>

<StepCard step={2} title="초안 받기">
Claude Code가 전체 구조와 초안을 작성합니다. 파일로 저장되므로 언제든 편집할 수 있습니다.
</StepCard>

<StepCard step={3} title="피드백으로 다듬기">
"도입부 더 감성적으로", "3번 문단 더 구체적으로" 처럼 부분적으로 수정 요청하세요.
</StepCard>

**실제 대화 예시:**

```
You: 교토 3일 여행기 블로그 글 써줘.
     첫날 기온 강변, 둘째날 후시미 이나리, 셋째날 아라시야마 다녀왔어.
     감성적인 톤으로, 사진 찍기 좋은 장소 위주로 소개해줘.

Claude: travel-kyoto.md 파일을 생성했습니다.

  # 교토 3일 — 붉은 도리이와 대나무 숲 사이에서
  
  교토역에서 내리자마자 공기가 달라진다...
  (1,400자 초안 작성 완료)

You: 도입부 더 짧게. 그리고 각 날짜마다 
     "추천 방문 시간" 항목 추가해줘.

Claude: travel-kyoto.md 수정했습니다. 도입부 3문장으로 줄이고
        각 섹션에 추천 방문 시간 추가했습니다.
```

<Callout type="tip">
`/writing` 스킬이 있다면 더 정교한 글쓰기 워크플로우를 실행할 수 있습니다.
</Callout>

## 웹사이트 만들기

코딩 지식 없이도 나만의 페이지를 만들 수 있습니다. 소개 페이지, 포트폴리오, 링크 모음 페이지 등에 활용하세요.

<StepCard step={1} title="원하는 것 설명하기">
어떤 페이지인지, 어떤 정보가 들어가야 하는지, 느낌이나 색상 선호도를 말해주세요.
</StepCard>

<StepCard step={2} title="파일 생성 확인">
Claude Code가 HTML/CSS 파일을 만들고 로컬 서버를 실행합니다. 브라우저에서 바로 확인할 수 있습니다.
</StepCard>

<StepCard step={3} title="보면서 수정하기">
화면을 보면서 "폰트 더 크게", "버튼 색 바꿔줘", "섹션 하나 추가해줘"처럼 실시간으로 수정하세요.
</StepCard>

**실제 대화 예시:**

```
You: 내 포트폴리오 페이지 만들어줘.
     이름: 박서연, 직업: 프리랜서 일러스트레이터
     작업물 갤러리 섹션, 연락처(인스타그램, 이메일) 포함.
     전체적으로 따뜻하고 아늑한 분위기.

Claude: portfolio/ 폴더에 파일 3개 생성했습니다.
  ✓ index.html
  ✓ style.css  
  ✓ main.js
  ✓ 로컬 서버 시작 → http://localhost:3000

You: 배경이 너무 밝아. 크림색 계열로 바꿔줘.
     그리고 갤러리 카드에 hover 효과 넣어줘.

Claude: style.css 수정 완료.
        배경 #FFF8F0으로 변경, 갤러리 카드에 
        위로 살짝 뜨는 hover 효과 추가했습니다.
```

<Callout type="tip">
완성된 페이지를 인터넷에 올리고 싶다면 "Vercel에 배포해줘"라고 하면 됩니다. 무료로 도메인까지 생성됩니다.
</Callout>

## 파일 정리 · 자동화

반복적이고 귀찮은 파일 작업을 한 번에 처리합니다.

<StepCard step={1} title="현재 상태 설명하기">
어떤 파일들이 있고, 어떻게 정리하고 싶은지 말해주세요.
</StepCard>

<StepCard step={2} title="실행 전 계획 모드">
`Shift+Tab`으로 **Plan Mode**에 진입하거나 요청 앞에 `/plan`을 붙이세요. Plan Mode에서는 Claude가 실제 변경 없이 계획만 세워 보여줍니다. 파일을 잘못 이동하면 되돌리기 어렵습니다.
</StepCard>

<StepCard step={3} title="실행 후 확인">
작업 완료 후 결과를 꼭 확인하세요. 문제가 있으면 `/rewind`로 되돌릴 수 있습니다.
</StepCard>

**실제 대화 예시:**

```
You: /plan 바탕화면 파일을 사진/, 문서/, 기타/로 자동 분류해줘.

Claude: ## 실행 계획 (Plan Mode)
  1. 바탕화면 파일 목록 분석
  2. jpg/png/heic → 사진/ 폴더로 이동
  3. pdf/docx/xlsx/pptx → 문서/ 폴더로 이동
  4. 나머지 → 기타/ 폴더로 이동
  5. 폴더 없으면 자동 생성

  총 47개 파일 처리 예정입니다. 승인하시겠어요?

You: 응 실행해줘.

Claude: ✓ 사진/ 23개, 문서/ 18개, 기타/ 6개 이동 완료.
```

<Callout type="warn">
파일을 이동·삭제하는 작업은 되돌리기 어렵습니다. 반드시 `/plan`으로 계획을 먼저 확인하세요.
</Callout>

## 안전하게 작업하기

실수가 걱정될 때 쓸 수 있는 안전장치입니다.

| 상황 | 방법 |
|------|------|
| 방금 한 작업 취소 | `Esc` 두 번 또는 `/rewind` |
| 지금 상태 저장 | `!git commit -m "저장"` |
| 이전 버전으로 돌아가기 | `!git log` 후 원하는 시점 선택 |
| 큰 작업 전 체크포인트 | `!git commit -m "작업 전 백업"` |

<Callout type="warn">
외부 서비스(이메일 발송, SNS 업로드, 결제 등)는 `/rewind`로 되돌릴 수 없습니다. 이런 작업은 실행 전에 꼭 한 번 더 확인하세요.
</Callout>

## 개발자용: 코드 리뷰

파일을 `@` 기호로 참조하면 Claude Code가 코드를 분석하고 개선점을 제안합니다.

```
You: @src/auth.js 이 파일 리뷰해줘.

Claude: auth.js 분석 결과:

  1. 5번 줄: 에러 처리 없음 → try-catch 추가 권장
  2. 12번 줄: await 누락 (비동기 처리 오류)
  3. 성능: 전체 배열 매번 탐색 → Map 구조 추천

  수정 적용할까요?

You: 1, 2번만 적용해줘.

Claude: auth.js 수정 완료했습니다.
```
