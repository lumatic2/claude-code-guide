# Claude Code 치트시트

강의 중·후 즉시 참조 가능한 완전한 명령어 레퍼런스입니다.

## 내장 슬래시 명령어

자주 쓰는 Claude Code 내장 명령어입니다. 전체 목록은 입력창에 `/`를 입력하거나 [공식 문서](https://code.claude.com/docs/en/commands)에서 확인할 수 있습니다.

| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 및 명령어 목록 |
| `/clear` | 대화 컨텍스트 완전 초기화 (새 대화 시작) |
| `/compact` | 긴 대화 요약 압축 (컨텍스트 유지) |
| `/context` | 현재 컨텍스트 사용량 시각화 |
| `/model` | 모델 변경 (Sonnet, Opus, Haiku 등) |
| `/plan [설명]` | Plan Mode 진입 — 계획만 세우고 변경은 차단 |
| `/rewind` | 이전 상태로 되돌리기 (대화·코드 체크포인트) |
| `/fast` | Fast Mode 토글 (Opus 4.6 전용) |
| `/effort` | 응답 품질·추론 effort 조정 (low~max) |
| `/cost` | 현재 세션 토큰·비용 확인 |
| `/permissions` | 허용/차단 규칙 UI 열기 |
| `/memory` | CLAUDE.md 편집·auto-memory 설정 |
| `/config` | 설정 UI (테마·모델·에디터 모드) |
| `/review [PR]` | PR 로컬 리뷰 |
| `/security-review` | 현재 브랜치 변경사항 보안 검토 |
| `/exit` | 세션 종료 |

<Callout type="tip">
`/drawing`, `/writing` 같은 명령은 내장이 아니라 **스킬**입니다. `~/.claude/skills/`에 직접 만들거나 플러그인으로 설치해서 추가합니다.
</Callout>

## 단축키

| 단축키 | 동작 |
|--------|------|
| `!` | 셸 명령 실행 (예: `! git status`) |
| `@파일명` | 파일 직접 참조 (예: `@README.md`) |
| `Shift+Tab` | 권한 모드 순환 (default → acceptEdits → plan …) |
| `Ctrl+C` | 현재 작업 중단 |
| `Ctrl+D` | 세션 종료 |
| `Ctrl+B` | 현재 작업 백그라운드로 보내기 |
| `Ctrl+L` | 프롬프트 입력 지우기 + 화면 재그리기 |
| `Ctrl+R` | 명령 히스토리 검색 |
| `Ctrl+T` | 할일(todos) 목록 토글 |
| `↑` / `↓` | 이전/다음 명령 탐색 |
| `Esc` | 현재 입력 취소 |

## 유용한 프롬프트 패턴 (10개)

| 상황 | 패턴 |
|------|------|
| 파일 수정 | "@파일명 X를 Y로 바꿔줘" |
| 버그 찾기 | "@파일명 이 파일에서 버그를 찾아줘" |
| 기능 추가 | "A 기능을 B에 추가해줘. 단계별로 설명해줘" |
| 디버깅 | "이 에러가 뭘 의미하는지 설명해줘: [에러 메시지]" |
| 코드 리뷰 | "@파일명 이 코드 검토해줘" |
| 설명 요청 | "@파일명 어떻게 동작하는지 설명해줘" |
| 테스트 작성 | "@파일명 X 함수에 대한 유닛 테스트 작성해줘" |
| 변경 전 계획 | "/plan으로 진입한 후 변경 계획을 세워줘" |
| 안전한 작업 | "/rewind 포인트를 설정한 후 변경 시작해줘" |
| 복잡한 작업 | "/plan으로 진입한 후 X를 구현해줘" |

## 설정 파일 위치

| 설정 | 경로 |
|------|------|
| 전역 설정 | `~/.claude/settings.json` |
| 전역 규칙 | `~/.claude/CLAUDE.md` |
| 프로젝트 설정 | `./CLAUDE.md` |
| 키보드 단축키 | `~/.claude/keybindings.json` |

## 워크플로우 팁

<Callout type="tip" title="커밋으로 체크포인트 설정">
큰 변경 전에 `git commit`으로 안전한 지점을 만들고, 필요시 `/rewind`로 돌아가세요.
</Callout>

<Callout type="tip" title="범위 명시로 정확도 향상">
"전체 프로젝트" 대신 `@파일명`으로 구체적 파일을 지정하면 더 정확한 도움을 받습니다.
</Callout>

<Callout type="tip" title="복잡한 작업은 /plan 모드 사용">
기능 구현, 대규모 리팩토링은 `/plan` 모드에서 계획을 먼저 세운 후 진행하세요.
</Callout>

<Callout type="tip" title="길어진 대화는 /compact로 압축">
같은 이슈에서 오래 작업하면 `/compact`로 대화를 정리하고 새로 시작하세요.
</Callout>

<Callout type="tip" title="모델 선택으로 효율성 향상">
단순 수정은 Haiku, 복잡한 설계는 Opus, 일반적 작업은 Sonnet을 `/model`로 선택하세요.
</Callout>
