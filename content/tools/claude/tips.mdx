# 실전 팁 · FAQ

오래 쓰면서 겪는 실전 상황들. 한 번씩 읽어두면 시간이 크게 절약됩니다.

## 비용 관리

Claude Code는 모델에 따라 토큰 단가가 다릅니다. 단순 작업에 Opus를 쓰면 금세 비용이 올라갑니다.

| 작업 성격 | 추천 모델 |
|---|---|
| 단순 수정·문자열 교체 | Haiku |
| 일반 코딩·글쓰기 | Sonnet |
| 복잡한 설계·리팩토링 | Opus |

```bash
/cost     # 현재 세션의 토큰·비용 확인
/model    # 모델 변경
/usage    # 플랜 한도·rate limit 확인
```

<Callout type="tip">
같은 세션에서 모델을 섞어 써도 됩니다. 설계는 Opus, 구현은 Sonnet, 자잘한 정리는 Haiku 식으로.
</Callout>

## 컨텍스트 관리

대화가 길어지면 응답이 느려지고 비용이 오릅니다. 오래된 맥락이 새 응답을 오염시키기도 합니다.

| 명령어 | 언제 쓰나요? |
|---|---|
| `/context` | 현재 얼마나 차 있는지 확인 |
| `/compact [지침]` | 대화를 요약해 컨텍스트만 줄이기 (주제 유지) |
| `/clear` | 완전히 새 대화 시작 (주제 전환) |
| `/rewind` | 특정 지점으로 되감기 (파일도 함께) |

**기준**: `/context`로 70% 넘으면 `/compact`. 주제가 완전히 바뀌면 `/clear`. 실수로 되돌리고 싶으면 `/rewind`.

## 막혔을 때 루틴

Claude가 엉뚱한 답만 반복하거나 에러가 계속 나면 다음 순서로:

1. **`/rewind`** — 마지막으로 정상이던 지점으로 돌아간다.
2. **요청을 쪼갠다** — "전체 리팩토링" → "이 함수만 먼저".
3. **`@파일명`으로 범위를 좁힌다** — 전체 프로젝트 대신 파일 하나만 참조.
4. **Plan Mode(`Shift+Tab` 또는 `/plan`)** — 실행 전에 계획만 받아 검토.
5. **`/clear`** — 컨텍스트가 오염된 것 같으면 새 대화로.

<Callout type="warn">
같은 지시를 세 번 이상 반복해도 안 되면 **문제 정의부터 틀렸을 가능성**이 큽니다. "왜 이게 안 되는 것 같아?"처럼 Claude에게 원인 분석을 먼저 요청하세요.
</Callout>

## 안전장치

```bash
!git commit -m "작업 전 백업"   # 큰 변경 전 체크포인트
!git diff                       # 변경 내용 확인
/rewind                         # Claude 세션 내 되돌리기
```

파일 삭제·외부 서비스 호출(이메일·업로드·결제)은 **`/rewind`로도 되돌릴 수 없습니다**. 실행 전 `/plan`으로 계획을 받아 확인하세요.

## 자주 받는 질문

**Q. 대화 기록은 어디에 저장되나요?**
세션은 `/resume`으로 다시 열 수 있습니다. 완전히 삭제하려면 `/clear`로 새 대화를 시작한 뒤 이전 세션을 종료하세요.

**Q. 인터넷 연결이 필요한가요?**
네, Claude Code는 Claude API를 호출하므로 인터넷이 필요합니다. 로컬 파일 작업도 모델 추론은 서버에서 일어납니다.

**Q. Windows에서만 쓸 수 있나요?**
아니요. macOS·Linux·Windows 모두 지원합니다. 이 가이드의 스크린샷은 Windows 기준입니다.

**Q. 더 자세한 문서는 어디서 보나요?**
공식 문서: [code.claude.com/docs](https://code.claude.com/docs). 단축키·명령어 변경은 여기서 먼저 공지됩니다.
