# 첫 사용법

> 설치가 끝났다면, 앱별로 한 가지씩만 직접 해보세요. 30분이면 감이 잡힙니다.

<Callout type="tip">
모든 작업은 **사이드바에 자연어 한국어로 요청**하면 됩니다. 명령어·단축키·문법을 외울 필요 없어요.
</Callout>

## Excel — 첫 프롬프트 5개

Excel을 열고 아무 데이터(거래내역, 매출, 인구 통계 등)를 띄워둔 채 사이드바에 입력해보세요.

### 1. 시트 구조 파악

```
이 시트가 어떤 데이터인지 한 단락으로 요약해줘.
```

### 2. 수식 설명

```
지금 선택한 셀의 수식이 뭘 계산하는지, 어떤 셀을 참조하는지 알려줘.
```

### 3. 새 분석 시트 만들기

```
"매출" 시트 데이터로 월별·카테고리별 합계를 정리한 새 시트를 만들어줘.
원본은 건드리지 말고.
```

### 4. 가정값 바꾸기

```
B2 셀의 할인율을 8%에서 12%로 바꾸고, 영향받는 셀만 알려줘.
```

### 5. 차트 추가

```
"월별 합계" 표로 라인 차트를 만들어서 같은 시트에 넣어줘. 
제목은 "2026년 분기별 매출 추이".
```

## Word — 첫 프롬프트 5개

작성 중인 문서를 열어둔 채 시도하세요.

### 1. 어조 다듬기

```
이 문서 전체를 격식있는 어조로 다듬어줘. 변경 추적(track changes)을 켠 채로.
```

### 2. 요약 만들기

```
이 보고서를 5줄짜리 임원용 요약(executive summary)으로 줄여서 맨 앞에 추가해줘.
```

### 3. 댓글 반영

```
3페이지에 달린 댓글들을 모두 반영해서 해당 단락만 다시 써줘.
```

### 4. 스타일 통일

```
모든 제목을 "Heading 2" 스타일로, 본문은 "Normal"로 통일해줘.
```

### 5. 표 만들기

```
2~4페이지에 흩어진 숫자들을 모아서, 항목·금액·비고 3열짜리 표로 정리해줘.
```

## PowerPoint — 첫 프롬프트 5개

빈 프레젠테이션 또는 작업 중인 덱을 열어두세요.

### 1. 템플릿대로 슬라이드 생성

```
지금 열려 있는 템플릿을 그대로 써서, 다음 5장으로 구성된 슬라이드를 만들어줘:
1. 표지 (제목: 2026 1분기 실적)
2. 매출 요약
3. 카테고리별 비중
4. 전년 대비 성장률
5. 다음 분기 전망
```

### 2. 표를 차트로

```
3번 슬라이드의 표를 네이티브 가로 막대 차트로 바꿔줘.
```

### 3. 슬라이드 정리

```
선택한 슬라이드의 폰트 크기·여백·정렬을 일관되게 다듬어줘. 내용은 그대로.
```

### 4. 발표자 노트 자동 작성

```
모든 슬라이드에 1분짜리 발표자 노트를 추가해줘. 한국어로, 청중은 임원진.
```

### 5. 다이어그램 만들기

```
"고객 여정" 슬라이드에 인지 → 관심 → 구매 → 재구매 4단계 가로 흐름도 다이어그램을 추가해줘.
```

## 잘 쓰는 요령

### 좋은 요청의 공통 패턴

```
[무엇을] [어떤 범위에] [어떤 제약으로]
```

예:
- ❌ "정리해줘" — 무엇을, 어디까지인지 모호
- ✅ "**A2:F100 범위**의 빈 셀을 **0으로** 채우고 **결과만 새 시트에** 넣어줘"

### 변경 전에 확인받기

큰 변경 전엔 한 번 검토받는 습관이 안전합니다.

```
바꾸기 전에 어떻게 바꿀 건지 먼저 알려줘. 동의하면 진행해.
```

### 망쳤을 때

`Ctrl+Z`로 되돌릴 수 있습니다. 사이드바에서도 "방금 한 거 되돌려줘"가 통하지만, **파일에 직접 가해진 변경은 Office의 실행 취소가 가장 확실**합니다.

<Callout type="warn">
Claude는 **현재 열려 있는 파일**만 보고 편집합니다. 닫혀 있는 파일·로컬 폴더의 다른 문서는 못 봅니다.
</Callout>
