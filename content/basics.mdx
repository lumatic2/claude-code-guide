# 핵심 사용법

## 슬래시 명령어

| 명령어 | 설명 |
|--------|------|
| `claude` | 대화형 세션 시작 |
| `claude "질문"` | 단일 질문 후 종료 |
| `/help` | 도움말 표시 |
| `/clear` | 컨텍스트 초기화 |
| `/compact` | 컨텍스트 압축 (긴 대화 시 토큰 절약) |
| `/plan` | 계획 모드 진입 (실행 전 계획 수립) |
| `/rewind` | 이전 상태로 되돌리기 |
| `/fast` | 빠른 응답 모드 (Opus → Sonnet 전환) |
| `/cost` | 현재 세션 비용 확인 |
| `/effort` | 응답 품질 조절 (low / normal / high) |
| `/model` | 모델 변경 |
| `/exit` | 종료 |

## 터미널 단축키

| 단축키 | 동작 | 예시 |
|--------|------|------|
| `!명령어` | 셸 명령 직접 실행 | `!git status` 또는 `!npm test` |
| `@파일경로` | 파일 내용을 컨텍스트에 포함 | `@src/index.ts 이 파일 리팩토링해줘` |
| `Shift+Tab` | 권한 모드 전환 | 승인 불필요하게 자동 실행 |
| `Ctrl+B` | 작업 백그라운드 실행 | 긴 빌드 작업 중 다른 요청 처리 |
| `Ctrl+C` | 현재 작업 중단 | 오래 걸리는 작업 즉시 중단 |
| `Esc × 2` | 직전 상태로 롤백 | 실수했을 때 이전 상태 복구 |

<Callout type="tip" title="! 접두사로 터미널 명령 실행">
셸 명령을 Claude Code 세션 안에서 바로 실행할 수 있습니다.
`!git log --oneline -5` → 최근 커밋 확인 후 바로 작업 지시로 이어지는 흐름이 자연스럽습니다.
</Callout>

<Callout type="tip" title="@ 접두사로 파일 참조">
여러 파일을 한 번에 참조할 수도 있습니다.
`@src/api.ts @src/types.ts 이 두 파일 타입 정의를 맞춰줘`
</Callout>

<Callout type="warn" title="Shift+Tab 자동 승인은 신중하게">
`Shift+Tab`을 누르면 모든 파일 수정을 자동 승인합니다. 예기치 않은 변경이 발생할 수 있으므로 
대규모 작업 전에는 사용하지 마세요. 꼭 필요한 반복 작업에만 사용하세요.
</Callout>

## 실습 예제

<StepCard step="1" title="파일 생성 요청">

**상황**: 간단한 스크립트가 필요할 때

```
> hello.py 파일 만들고 Hello World 출력하는 코드 작성해줘
```

Claude Code가 파일을 생성하고 코드를 작성합니다.

</StepCard>

<StepCard step="2" title="버그 수정">

**상황**: 실행 중 오류가 발생했을 때

```
> app.js의 TypeError 고쳐줘
> 에러: TypeError: Cannot read properties of undefined (reading 'map')
> at app.js:42
```

에러 메시지를 함께 붙여넣으면 더 정확하게 수정합니다.

</StepCard>

<StepCard step="3" title="파일 참조 + 설명 요청">

**상황**: 기존 코드가 어떻게 동작하는지 알고 싶을 때

```
> @src/utils/auth.ts 이 파일 어떻게 동작하는지 설명해줘
```

`@` 접두사로 파일을 참조하면 전체 내용을 컨텍스트에 포함시킵니다.

</StepCard>

<StepCard step="4" title="범위 지정 구현">

**상황**: 구체적인 요구사항을 만족하는 코드가 필요할 때

```
> app/api/users.ts에서 인증 미들웨어 추가해줘.
> JWT 토큰 없으면 401 반환하고, 있으면 req.user에 디코딩된 정보 넣어줘
```

목표 + 파일 + 완료 기준을 한 번에 명시하면 품질이 안정됩니다.

</StepCard>

<StepCard step="5" title="계획 수립 후 구현">

**상황**: 복잡한 프로젝트를 체계적으로 진행하고 싶을 때

```
> /plan
> 할 일 관리 앱을 React로 만들어줘.
> 기능: 할 일 추가, 완료 표시, 삭제
```

`/plan` 모드에서 구조를 먼저 확인하고 진행합니다.

</StepCard>

<StepCard step="6" title="컨텍스트 관리">

**상황**: 대화가 길어져서 응답이 느려질 때

```
> /compact
```

대화 내용을 요약해서 컨텍스트 윈도우를 확보합니다.
작업이 끝났거나 주제가 바뀌면 `/clear`로 완전 초기화하세요.

</StepCard>

## 효과적인 프롬프트 패턴

| 상황 | 패턴 |
|------|------|
| 기능 추가 | `[파일]에서 [기능] 추가해줘. [완료 기준]` |
| 버그 수정 | `[파일]의 [에러] 고쳐줘: [에러 메시지]` |
| 리팩토링 | `[파일] 리팩토링해줘. [제약: 기능 변경 없이]` |
| 테스트 | `[함수]에 대한 유닛 테스트 작성해줘` |
| 코드 리뷰 | `@[파일] 코드 리뷰해줘. 보안 취약점 위주로` |

<Callout type="warn" title="항상 git commit 먼저">
Claude Code는 실제 파일을 수정합니다. 큰 변경 전에는 반드시 `!git commit`으로 체크포인트를 만들어두세요.
실수했다면 `Esc × 2` 또는 `/rewind`로 롤백할 수 있습니다.
</Callout>

<Callout type="tip" title="비용과 속도 조절">
간단한 수정은 `/effort low` 또는 `/fast`로 빠르게 처리하고,
복잡한 설계는 `/effort high`로 깊은 추론을 통해 해결합니다.
`/cost`로 현재 세션 비용을 확인할 수도 있습니다.
</Callout>
