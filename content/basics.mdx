# 기본 사용법

## 슬래시 명령어

대화 중 `/`로 시작하는 명령어로 Claude Code의 기능을 제어합니다.

| 명령어 | 설명 | 언제 쓰나요? |
|--------|------|-------------|
| `/help` | 도움말 보기 | 모르는 게 생겼을 때 |
| `/clear` | 대화 맥락 초기화 | 새로운 주제로 넘어갈 때 |
| `/plan` | 계획 모드 | 큰 작업 시작 전 단계 확인 |
| `/compact` | 대화 압축 | 대화가 너무 길어졌을 때 |
| `/rewind` | 직전 작업 취소 | 결과가 마음에 안 들 때 |
| `/cost` | 사용 비용 확인 | 비용이 궁금할 때 |
| `/model` | AI 모델 변경 | 더 빠르거나 강력한 모델로 전환 |
| `/exit` | 종료 | 작업 마쳤을 때 |

<Callout type="tip">
`/plan`은 습관처럼 쓰세요. 특히 파일을 많이 건드리는 작업이라면 실행 전에 계획을 먼저 보는 게 안전합니다.
</Callout>

## 터미널 단축키

| 단축키 | 동작 |
|--------|------|
| `!명령어` | 터미널 명령 직접 실행 (예: `!ls`, `!git status`) |
| `@파일경로` | 파일 내용을 대화에 포함 (예: `@내문서.txt 이 내용 요약해줘`) |
| `Shift+Tab` | 권한 모드 전환 (매번 확인 → 자동 실행) |
| `Ctrl+U` | 입력 중인 텍스트 전체 삭제 |
| `Esc` × 2 | 진행 중인 작업 취소 |

## 좋은 요청 작성법

Claude Code는 지시가 명확할수록 더 좋은 결과를 냅니다.

### 구체적으로 말하기

| 모호한 요청 | 구체적인 요청 |
|------------|-------------|
| "웹사이트 만들어줘" | "내 소개 페이지 만들어줘. 이름, 직업, 연락처 포함. 미니멀한 디자인으로." |
| "글 써줘" | "교토 여행기 블로그 글 써줘. 3박 4일, 감성적인 톤, 800자 내외." |
| "정리해줘" | "Downloads 폴더에서 1년 넘은 파일들만 Archive 폴더로 옮겨줘." |

### 맥락 주기

결과물이 어디에 쓰일지 알려주면 더 적합한 결과가 나옵니다.

```
"인스타그램 피드용 카드뉴스 텍스트 써줘. 
 주제는 '번아웃 극복 5가지 방법', 2030 직장인 대상."
```

### 단계별로 나누기

한 번에 너무 많이 요청하면 결과가 엇나갈 수 있습니다.

```
# 좋지 않은 예
"웹사이트 만들고 SEO 최적화하고 배포까지 해줘"

# 좋은 예
1단계: "소개 페이지 HTML 먼저 만들어줘"
2단계: (확인 후) "SEO 메타태그 추가해줘"
3단계: (확인 후) "Vercel에 배포해줘"
```

<Callout type="tip">
막히면 "어떻게 하면 좋을까?"라고 물어보세요. Claude Code가 방향을 제안해줍니다.
</Callout>

### 실습 1: 나만의 소개 페이지

<StepCard step={1} title="Claude Code 실행">
터미널을 열고 `claude`를 입력해 세션을 시작합니다.
</StepCard>

<StepCard step={2} title="요청하기">
아래를 그대로 입력하거나 내용을 바꿔서 사용하세요.

```
내 소개 페이지 HTML 파일 만들어줘.
이름: 홍길동
직업: 프리랜서 디자이너
소개 한 줄: "공간과 경험을 디자인합니다"
연락처: hong@gmail.com
깔끔한 흰색 배경, 가운데 정렬로 만들어줘.
```
</StepCard>

<StepCard step={3} title="결과 확인 후 수정">
생성된 `index.html` 파일을 브라우저에서 열어 확인합니다. 마음에 안 드는 부분은 바로 수정 요청하세요.
</StepCard>
