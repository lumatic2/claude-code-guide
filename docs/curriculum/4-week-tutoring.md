# 4주 AI 실무 과외

> 주 1회 2시간 × 4주. 한 명~소수 대상.
> **목표**: 4주 후 학습자가 **선생님 없이 새 도구·모델·기법이 나와도 혼자 평가하고 자기 업무에 통합할 수 있는 사람**이 된다.

## 왜 이 구조인가

AI 산업은 6개월마다 패러다임이 바뀐다. "이거 외워라" 식의 콘텐츠는 4주가 끝나는 시점에 이미 부분 노후화한다. 그래서 이 과외는 **콘텐츠 전수가 아니라 사고 과정 전수**다.

4주 동안 학습자는 같은 학습 사이클을 4번 돌려본다:
1. **첫 모델 잡기** — 지금 시점의 이해를 만들고 갱신하는 법
2. **정보 시스템 구축** — 매주 신호를 얻는 자기 루틴
3. **평가 프레임워크** — 새 것이 쓸만한지 30분 안에 판단
4. **실험·통합 사이클** — 평가 통과한 것을 자기 워크플로우에 녹이기

4주 후 학습자는 "이번에 ○○라는 게 나왔다"는 소식이 들리면 본능적으로: **신호인지 판단 → 30분 평가 → 통합/대기/탈락 결정** 사이클을 돈다. 그게 진짜 산출물이다.

## 진행 원칙

- **학습자 질문에서 시작**: 매 차시 첫 행동은 학습자가 가져온 자기 업무 맥락의 질문 1개. 강의자가 정해서 가르치지 않는다.
- **사고 과정을 시연**: 강의자는 답을 주지 않고, **답을 찾아가는 과정**을 눈앞에서 보여준다. 어떤 자료를 어떻게 검색하는지, 어떻게 검증하는지, 막혔을 때 어떻게 빠져나오는지.
- **"모른다"를 자주 보여줄 것**: 강의자가 "이 부분은 나도 잘 모름. 같이 찾아보자"를 자주 해야 학습자가 "모르는 게 정상이고 찾아가는 거구나"를 체득한다. 권위적인 강의는 의존성만 키운다.
- **그 자리에서 한 번, 일주일 후 한 번**: 차시 안에서 학습자가 같은 사이클을 즉석에서 한 번 돌리고, 일주일간 본인 업무에서 한 번 더 돌린다.
- **차시 첫 30분은 지난 주 회고**: 잘된 것보다 막힌 것이 더 중요. 막힌 지점이 다음 차시의 새 질문이 된다.

## 차시 4단 구조

매 차시 같은 4단으로 진행:

1. **이번 주 질문** (15분) — 학습자가 가져온 자기 업무 맥락 질문 1개. 강의자는 그 질문을 더 좋은 질문으로 다듬어 줌.
2. **함께 탐색** (60분) — 강의자가 사고 과정을 시연. 자료를 어디서 찾고, 어떻게 비교하고, 어떻게 검증하는지. 학습자는 옆에서 같이 본다.
3. **혼자 한 번 더** (30분) — 학습자가 즉석에서 같은 패턴으로 자기 다른 질문 1개를 처리. 강의자는 옆에서 "지금 어디가 모호한가"만 짚는다.
4. **다음 주까지 자기 사이클** (15분) — 1주 동안 본인 업무에 같은 패턴 적용 + 회고 메모 가져오기로 합의.

각 차시 첫 30분은 지난 주 사이클 회고가 별도로 들어가니, 사실 진짜 신규 콘텐츠는 90분이다. 분량 욕심 내지 말 것.

## 1주차: 첫 모델 잡기 — "지금 AI를 어떻게 봐야 하는가"

### 학습자가 얻는 메타-스킬
빠르게 변하는 분야의 **첫 이해를 어떻게 만들고**, 그게 **틀릴 수 있다는 걸 받아들이고**, **갱신하는 법**.

### 이번 주 질문 (예시)
- "왜 AI가 이런 데서 헛소리해요?"
- "Claude랑 GPT 뭐가 다른지 모르겠어"
- "프롬프트 잘 쓰는 법은 진짜 있는 거예요?"

### 함께 탐색

**(1) 강의자의 현재 멘탈 모델 공개 (20분)**
강의자가 자기 머릿속의 AI 모델을 그림으로 그려가며 보여준다:
- LLM = 다음 토큰 예측기 (그래서 환각이 디폴트)
- 컨텍스트 = 모델이 답할 때 보고 있는 모든 것
- 도구 사용 = 모델이 외부 시스템에 손 뻗는 메커니즘
- 에이전트 = LLM + 도구 + 루프
- 하네스 = 에이전트가 살아 움직이는 환경

**핵심**: "이게 정답이라고 외우지 마세요. 제가 지금 이 시점에 갖고 있는 모델일 뿐이에요. 6개월 후엔 바뀝니다."

**(2) 이 모델이 어디서 왔는지 보여주기 (20분)**
- Anthropic 공식 블로그 (Building Effective Agents 등)
- Karpathy "Intro to LLMs" YouTube 1시간
- OpenAI/Anthropic 공식 문서
- Simon Willison 블로그
강의자가 "여기서 이걸 봐서 이렇게 정리했다"를 실제 화면 켜고 보여준다. 학습자에게 **1차 출처를 직접 보는 습관**을 심는 게 목적.

**(3) 학습자 질문을 이 모델로 함께 분석 (15분)**
"왜 헛소리해요?" → "다음 토큰 예측기라는 본질에서 시작해보면..." 식으로 모델을 적용.

**(4) 모델의 한계 솔직히 보여주기 (5분)**
"이 부분은 나도 아직 잘 모름. 다음에 더 찾아볼 영역."

### 혼자 한 번 더
학습자가 자기 다른 의문 1개를 골라 같은 패턴으로 분석:
- 이 질문에 답하려면 어떤 자료를 봐야 하나?
- 어디서 시작해야 하나?
- 답을 찾으면 어떻게 검증하나?

강의자는 답을 주지 말 것. "그 자료를 어디서 찾을 것 같아요?"만 묻기.

### 다음 주까지 자기 사이클
- 평소 헷갈렸던 AI 관련 질문 3개 적기
- 각자 1차 출처(공식 문서·블로그·발표 영상)에서 답 찾아 1문장으로 정리
- **막혔던 질문 1개**를 다음 주에 가져오기

## 2주차: 정보 시스템 구축 — "어디서 신호를 얻을 것인가"

### 학습자가 얻는 메타-스킬
매주 **30분 이내에 최신성을 유지**하는 자기만의 정보 루틴.

### 이번 주 질문 (예시)
- "AI 뉴스가 너무 많아서 뭘 봐야 할지 모르겠어요"
- "정말 중요한 변화는 어떻게 알아채요?"
- "유튜브 추천만 보면 되나요?"

### 함께 탐색

**(1) 강의자의 정보 수집 시스템 공개 (20분)**
화면 켜고 실제로 보여준다:
- X (Twitter) 팔로우 리스트 — 누구를 왜 팔로우하는지
- RSS / Substack 구독 — 어떤 뉴스레터를 왜
- YouTube 채널 — 어떤 사람이 어떤 깊이로
- 공식 출처 — Anthropic / OpenAI / Google 블로그
- 논문/연구 소스 — Arxiv 어떻게 거르는지
- Discord / 커뮤니티

**(2) 시그널 vs 노이즈 거르는 3축 (15분)**
- **출처 권위**: 1차 출처 (공식 발표·논문) > 2차 (해석·요약) > 3차 (반응·밈)
- **새로움**: 진짜 새 패러다임 vs 기존의 변종
- **자기 작업 관련성**: 내가 다음 1~3개월 안에 손댈 영역인가

이 3축을 종이에 그려놓고 학습자가 본 최근 뉴스 1개를 같이 채점해 본다.

**(3) 30분 주간 루틴 라이브 시연 (20분)**
강의자가 실제로 월요일 30분 루틴을 그 자리에서 돈다:
- 5분: 핵심 출처 빠르게 훑기 (북마크 폴더 1개)
- 15분: 눈에 들어온 것 2~3개 깊이 읽기
- 5분: 시도해볼 만한 것 1개 메모
- 5분: 다음 주에 평가할 것 후보 적기

**(4) 함정 짚기 (5분)**
- 인플루언서 트렌드 추종 vs 1차 출처
- "도구 카탈로그형 콘텐츠"의 함정 (다 좋아 보이지만 자기 손에 닿는 게 없음)
- 정보 과잉 → 마비. 30분 못 넘기게.

### 혼자 한 번 더
학습자가 그 자리에서:
- 자기 X 팔로우 / 뉴스레터 구독 한번 정리
- 30분 루틴을 짧은 버전(15분)으로 시도
- 그 결과 시도해볼 만한 것 1개 메모

### 다음 주까지 자기 사이클
- 1주일 동안 30분 루틴 3회 실행
- 발견한 것 중 시도해볼 후보 3개 정리
- **그중 1개를 다음 주 3주차 평가 대상으로 가져오기**

## 3주차: 평가 프레임워크 — "이게 내 업무에 쓸만한가"

### 학습자가 얻는 메타-스킬
새 도구/모델/기법이 나왔을 때 **30분 안에 "내 업무에 쓸만한가" 판단**하는 자기 루틴.

### 이번 주 질문 (예시)
- "2주차에 발견한 ○○ 도구, 진짜 좋은 건지 어떻게 알지?"
- "○○가 ○○보다 낫다는데 어떻게 비교해요?"
- "데모 영상은 멋진데 실제론 안 될까봐 못 시도하겠어요"

### 함께 탐색

**(1) 강의자의 30분 평가 프레임워크 공개 (20분)**
화면 켜고 실제로 한 번 돈다:

| 시간 | 단계 | 무엇을 보는가 |
|---|---|---|
| 5분 | **정의 확인** | 공식 문서 첫 페이지 + 데모 영상 1개. "정확히 뭘 하는 도구인가" 한 문장 요약 |
| 10분 | **내 손에 닿는 작업으로 시도** | 자기 업무 실제 작업 1개를 그걸로 처리 |
| 10분 | **기존 방식과 비교** | 같은 작업을 기존 방식으로 했을 때 vs 새 도구. 시간·품질·번거로움 |
| 5분 | **결정** | 통합 / 대기 / 탈락 — 그리고 왜 그 결정인지 한 문장 |

**핵심**: "정답이 아니에요. 30분 안에 결론 내는 게 핵심입니다. 1시간 넘어가면 평가가 아니라 학습이 돼버려요."

**(2) 학습자가 가져온 도구로 함께 평가 (25분)**
강의자가 학습자 옆에서 30분 프레임워크를 같이 돈다. 학습자가 손을 움직이고, 강의자는 "지금 단계에서 뭘 봐야 하는지"만 짚는다.

**(3) 함정 짚기 (10분)**
- **데모 영상의 마법**: 깔끔한 데모 vs 실제 손에서 안 됨. 항상 자기 작업으로 시도.
- **마케팅 vs 1주 사용 후 회고**: 평가는 1차 판단. 통합 결정은 1주 시도 후.
- **얼리어답터 함정**: 새 것은 다 좋아 보임. "내가 진짜 매일 쓸 것인가" 자문.
- **비교 군의 부재**: "X가 좋다"는 평가는 "Y에 비해 어떻게 좋다"가 없으면 의미 없음.

### 혼자 한 번 더
학습자가 본인이 들어본 **다른** 새 도구 1개를 골라 그 자리에서 30분 프레임워크 적용. 강의자는 시간 체크만.

### 다음 주까지 자기 사이클
- 1주 동안 새로운 것 2개를 30분 프레임워크로 평가
- 결정 기록 (통합/대기/탈락 + 이유)
- **통합 결정한 1개를 4주차 통합 대상으로 가져오기**

## 4주차: 실험·통합 사이클 — "내 워크플로우에 어떻게 녹일까"

### 학습자가 얻는 메타-스킬
평가 통과한 것을 자기 워크플로우에 **통합 → 운영 → 회고**하는 자기 사이클.

### 이번 주 질문 (예시)
- "3주차에 평가한 ○○를 어떻게 매일 쓸지?"
- "내 업무 어디까지 자동화할 수 있을까?"
- "한 번 쓰고 마는 게 아니라 루틴으로 만들려면?"

### 함께 탐색

**(1) 강의자의 통합 사례 시연 (25분)**
강의자가 자기 워크플로우 1개를 통합 → 운영 → 회고 → 개선한 실제 사례를 처음부터 끝까지 보여준다:
- 처음 평가 통과 시점의 메모
- 1~2주 수동 적용 기록
- 루틴화한 시점 (스킬/슬래시 커맨드/cron)
- 1개월 후 회고: 살아남았는지, 다듬었는지, 뺐는지

**(2) 통합의 3단계 (10분)**

| 단계 | 기간 | 무엇을 하는가 |
|---|---|---|
| **수동 적용** | 1~2주 | 의식적으로 매일 사용. 어색해도 일단 씀. 사용 기록 메모. |
| **루틴화** | 적용이 안정되면 | 슬래시 커맨드/스킬/cron/단축키로 묶음. 손이 자동으로 가게. |
| **회고·개선** | 1개월 후 | 잘 안 통하면 빼거나 다듬기. 완전히 사라지는 것도 정상. |

**(3) 학습자 통합 함께 설계 (20분)**
3주차에 통합 결정한 도구를:
- 어느 작업에 쓸지 구체적으로 정의
- 1~2주 수동 적용 기준 정하기 ("매일 ○○할 때마다 꼭 켠다")
- 루틴화 시점에 어떻게 묶을지 미리 그림 그리기

**(4) 함정 짚기 (5분)**
- **다 자동화하려는 욕심**: 1년에 2번 하는 일까지 자동화 → 자동화 비용이 더 큼
- **수동 단계 건너뛰기**: 평가 끝나자마자 바로 루틴화 → 안 맞는 거 굳혀버림
- **회고 안 하기**: 1개월 후 정말 쓰는지 안 보면 도구 무덤만 쌓임

### 혼자 한 번 더
학습자가 3주간 평가한 도구 중 1개를 자기 워크플로우에 즉석 통합:
- 슬래시 커맨드 또는 스킬 1개 만들기
- 또는 cron/자동화 1개 설정
- 다음 1주 운영 계획 적기

### 다음 4주까지 자기 사이클
**4주 사이클 1번 더 혼자 돌리기**:
1. 정보 루틴 4주 유지
2. 새 도구 4~5개 평가
3. 통합 결정 1~2개
4. 루틴화 시도

4주 후 결과 공유 (메일·메시지·짧은 문서).

## 부록: 4주 후

### 학습자가 가져가야 할 3가지
1. **자기 멘탈 모델** — 지금 시점 AI에 대한 본인 정리. 다음 6개월 동안 갱신할 베이스.
2. **30분 주간 루틴** — 정보 수집 + 평가 후보 발굴.
3. **30분 평가 프레임워크 + 통합 사이클** — 새 것이 나왔을 때 결정하는 자기 절차.

### 강의자가 점검할 것
- 4주 동안 학습자가 가져온 질문이 **점점 좋아졌는가** (모호 → 구체)
- 학습자가 강의자에게 의존하지 않고 **자기 1차 출처 검색**을 하는가
- 마지막 차시에 학습자가 "○○이라는 새 도구는 △△ 이유로 통합 안 함"이라고 **자기 판단으로 거절**할 수 있는가

세 번째가 가장 중요하다. 무조건 새 것을 받아들이는 게 아니라 **자기 판단으로 거절**할 수 있게 되는 것이 4주의 진짜 산출물.

### 학습자가 더 가고 싶다면
- 같은 분야 다른 사람들의 학습 시스템 비교 (강의자가 자기만 옳다고 하지 말 것)
- 본인 직군의 1차 정보 출처 직접 발굴 (의료·법률·교육 등)
- 자기 학습 사이클을 회고하는 월간 루틴 추가
