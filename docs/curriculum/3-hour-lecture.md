# 3시간 AI 실무 강의

> 단발 3시간 강의용. 50분 × 3교시 + 휴식 10분 × 2.
> 청중 5~50명, 비-개발자 포함 가능.
> **목표**: 청중이 **"월요일에 회사에서 시도해볼 것 1개"**를 들고 나간다.

## 4주 과외와의 관계

이 강의는 4주 과외의 압축본이 **아니다**. 목적이 다르다:

- **4주 과외** = 학습자가 스스로 평가·통합할 수 있는 사람이 되게 함 (메타-스킬)
- **3시간 강의** = 청중이 "이거 나도 써봐야겠다"는 충격과 첫 시도 1개를 들고 나가게 함 (감각 전수)

3시간으론 메타-스킬을 심는 게 불가능하다. 그래서 강의는 **충격 + 데모 + 액션 1개**에 집중한다.

## 강의자 사전 준비

- 자기 노트북 + Claude Code 설치된 실제 프로젝트 1개 (라이브 데모용)
- 백업: 데모 영상 1개 미리 녹화 (인터넷·계정·API 다 끊겨도 보여줄 수 있게)
- 슬라이드 구조: **개념 슬라이드 최소화**. 데모 화면이 슬라이드. 화면 글자 크기 18pt 이상.
- 청중 사전 안내: "노트북 가져오세요. Claude.ai 계정만 있으면 됩니다."

## 청중 사전 준비

- 노트북
- Claude.ai 무료 계정 (Pro/Max는 강의 중 권장만, 강요 X)
- 자기 업무 영역 1개를 머릿속에 가져올 것 ("뭘 자동화해보고 싶나")

## 차시 3단 구조

매 교시 같은 3단으로 진행:

| 시간 | 단계 | 강의자가 무엇을 하는가 | 청중이 무엇을 가져가는가 |
|---|---|---|---|
| 10분 | **개념** | 핵심 1~2개만 짧게 | "왜 이게 작동하는지" 한 문장 이해 |
| 30분 | **라이브 데모** | 실제 작동을 보여줌 | "와, 이게 되네"의 충격 |
| 10분 | **청중 액션 + Q&A** | 따라하기 또는 질문 받기 | 손에 잡히는 첫 시도 1개 |

## 강의 흐름 한눈에

| 교시 | 주제 | 청중이 얻는 것 |
|---|---|---|
| **1교시 (50분)** | AI는 어떻게 작동하나 + 프롬프트 핵심 | "왜 AI가 헛소리를 하는지", "어떻게 물어야 잘 답하는지" |
| **2교시 (50분)** | 컨텍스트와 에이전트 | "프롬프트만으로는 한계가 있고, 도구가 필요한 이유" |
| **3교시 (50분)** | Claude Code 데모 + 실무 적용 | "월요일에 시도해볼 1개" |

## 1교시 (50분): AI는 어떻게 작동하나 — 프롬프트 핵심

### 개념 (10분)

**오프닝 한 문장**: "ChatGPT, Claude, Gemini — 다 같은 본질이다. 다음에 올 단어를 한 번에 하나씩 예측하는 기계."

청중에게 던지는 질문 3개:
1. "AI가 자신 있게 거짓말 하는 거 본 적 있죠?" → 환각이 왜 디폴트인지 설명
2. "같은 질문에 다른 답이 오는 거 본 적 있죠?" → 확률적 생성
3. "긴 대화 중간에 앞 얘기 까먹는 거 본 적 있죠?" → 컨텍스트 윈도우

여기서 한 가지만 기억시키기: **AI는 그럴듯한 답을 만드는 기계지, 진실을 검색하는 기계가 아니다**.

### 라이브 데모 (30분) — 프롬프트 5단계 진화

같은 업무 질문을 5단계로:

```
v1 (막던지기): "고객 응대 메일 써줘"
v2 (+역할):    "당신은 우리 회사 CS팀 5년차. 고객 응대 메일 써줘"
v3 (+구조):    "사과 → 원인 → 해결책 3단 구성으로"
v4 (+예시):    "이런 식으로: [좋은 예시 1개]"
v5 (+반례):    "이런 사과는 피해주세요: [나쁜 예시]"
```

각 단계 결과를 **화면에 같이 띄워두고** "어디서 점프했는지" 짚는다.

핵심 메시지: "프롬프트는 마법 주문이 아니다. **모델에게 충분한 정보를 주는 일**일 뿐이다."

### 청중 액션 + Q&A (10분)

각자 자기 업무에서 자주 하는 작업 1개를 골라 v1 → v3까지 만들어 본다 (시간상 v5까지는 못 함). 짝꿍과 결과 비교 1분.

남은 시간 Q&A 자주 나오는 질문:
- "GPT랑 Claude 뭐가 더 좋아요?" → 작업 성격마다 다름. 둘 다 써보세요.
- "프롬프트 외워야 하나요?" → 외울 필요 없다. **구조**(역할·구조·예시·반례)만 머리에 두면 됨.
- "한국어로 해도 돼요?" → 됨. 영어가 약간 더 안정적이지만 차이 작아짐.

## 2교시 (50분): 컨텍스트와 에이전트 — 프롬프트 너머

### 개념 (10분)

**핵심 메시지**: "프롬프트만 잘 쓰는 것으론 한계가 빠르게 온다. 진짜 차이는 **컨텍스트**와 **도구**다."

3개 개념만 짚기:

**1. 컨텍스트 = 모델이 답할 때 보고 있는 모든 것**
프롬프트, 첨부 파일, 이전 대화, 시스템 지시 전부 포함. "프롬프트 엔지니어링"보다 "컨텍스트 엔지니어링"이 실무 핵심.

**2. RAG = 필요한 자료를 그때그때 가져다 주는 것**
"우리 회사 매뉴얼", "최근 뉴스" 같이 모델이 모르는 정보를 검색해서 컨텍스트에 넣는 패턴. 사내 챗봇·문서 Q&A 솔루션 다 이 구조.

**3. 에이전트 = LLM + 도구 + 루프**
한 번 답하고 끝나는 챗봇과 다르다. 목표 받으면 계획 → 도구 호출 → 결과 보고 → 다음 행동을 자율적으로 도는 것. Claude Code, Cursor 같은 게 다 에이전트.

### 라이브 데모 (30분) — 컨텍스트 차이 + 에이전트 첫 경험

**Part A (10분): 컨텍스트 있을 때 vs 없을 때**

같은 질문을 두 방식으로:
```
A. Claude.ai 새 대화: "내 회사 휴가 정책 정리해줘"
   → 일반론적인 헛소리

B. 같은 질문 + 회사 휴가 규정 PDF 첨부
   → 실제 회사 정책 정확히 정리
```

핵심: "모델이 똑똑해지는 게 아니라 **보고 있는 게 달라지는 것**".

**Part B (20분): Claude Code 첫 데모**

자기 프로젝트 1개로 라이브:
```
1. 프로젝트 폴더에서 claude 실행
2. CLAUDE.md 보여주기 (1분 — "이게 뭐고 왜 있는지")
3. 작은 변경 1개 요청: "이 화면에 다크모드 토글 추가해줘"
4. Plan Mode → 실행 → 파일 수정 보여주기
5. 끝나면 빌드 돌려서 결과 확인
```

청중이 처음 보는 거면 **충격이 큼**. 그 충격을 이용해서 "이게 가능한 이유는 모델이 도구(파일 읽기·쓰기·실행)를 쓸 수 있게 하네스가 깔려있기 때문"을 설명.

### 청중 액션 + Q&A (10분)

청중 2~3명에게 묻기: "방금 본 거를 자기 업무에 어떻게 응용할 수 있을 것 같나?" → 짧게 받고 강의자가 한두 줄씩 코멘트.

자주 나오는 Q&A:
- "코드 못 쓰는데 Claude Code 써도 돼요?" → 됨. 처음에는 사람한테 부탁하듯 자연어로.
- "보안은요? 회사 자료 올려도 돼요?" → 회사 정책 확인. Claude는 기본적으로 학습에 안 씀(Anthropic 정책), 그래도 민감 정보는 사내 정책 따를 것.
- "비싸지 않나요?" → Pro 월 $20부터. 무료로 충분히 시작 가능.

## 3교시 (50분): Claude Code 깊이 보기 + 실무 적용

### 개념 (10분)

**핵심 메시지**: "도구는 많다. 중요한 건 **자기 업무에 한 줄을 꽂는 것**."

3개 패턴 소개:

**1. 슬래시 커맨드 — 반복 작업의 한 줄 명령**
"매일 오전에 어제 한 일 요약해줘" 같은 걸 `/standup` 한 줄로 만들 수 있음.

**2. CLAUDE.md — 매번 설명 안 해도 되게**
프로젝트 컨벤션·기술 스택·주의사항을 한 번 적어두면 매 세션 자동 로드.

**3. 위임 vs 직접 — 의사결정 기준**
| 위임 OK | 사람이 해야 |
|---|---|
| 반복 + 결과 검증 쉬움 | 1회성 + 책임 큼 |
| 정형화된 출력 | 창의적 판단 |
| 실수해도 복구 가능 | 실수 = 큰 손실 |

### 라이브 데모 (30분) — 실무 워크플로우 한 사이클

자기 프로젝트로 처음부터 끝까지 시연:

```
1. 작업 정의 — "○○ 기능 추가" (PRD 한 단락)
2. Plan Mode — 모델이 계획 세우는 거 보여주기
3. 실행 — 파일 수정·실행·테스트 자동
4. 교차검증 — /codex 같은 다른 모델로 리뷰 (있으면)
5. 커밋 — git에 남기는 것까지
```

이게 흐름의 전부라는 걸 강조. **각 단계 사이에 사람의 검토가 들어가는 게 핵심**.

### 청중 액션 + Q&A (10분) — 클로징

각자 종이/노트에 쓰기:
1. **이번 주에 시도해볼 것 1개** (작게)
2. **막힐 것 같은 부분 1개** (지금 미리 적기)
3. **누구에게 도움 청할 것인가** (사람 1명)

짝꿍과 1분씩 공유 → 강의자가 2~3명 발표.

클로징 메시지 (3개 중 하나 골라):

**Option A — 행동 촉구형**:
> "오늘 들은 거 안 쓰면 일주일 안에 80% 잊습니다. 오늘 저녁에 한 번만 켜보세요. 그게 시작입니다."

**Option B — 관점 전환형**:
> "AI가 사람을 대체한다는 건 틀린 말입니다. **AI를 쓰는 사람이 안 쓰는 사람을 대체한다**는 게 맞습니다. 오늘이 그 시작이길 바랍니다."

**Option C — 실용형**:
> "제일 비싼 건 시간입니다. AI는 시간을 벌어주는 도구예요. 오늘 배운 거 하나만이라도 다음 주 업무에 적용하면, 이 강의 본전 뽑은 겁니다."
