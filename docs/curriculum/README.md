# AI 실무 커리큘럼

> Claude Code를 비롯한 AI 도구를 실무에 녹여 쓰는 사람을 만드는 커리큘럼.
> 외부 강의·개인 과외·자가학습용 단일 소스.

## 두 트랙 — 목적이 다르다

| 트랙 | 분량 | 목표 | 산출물 |
|---|---|---|---|
| **[4주 과외](./4-week-tutoring.md)** | 주 1회 2시간 × 4주 (총 8시간) | 학습자가 **선생님 없이** 새 도구·모델·기법을 평가·통합할 수 있는 사람이 되기 | 자기만의 정보 루틴 + 30분 평가 프레임워크 + 통합 사이클 |
| **[3시간 강의](./3-hour-lecture.md)** | 단발 3시간 (50분 × 3교시 + 휴식 10분 × 2) | 청중이 **충격받고** "이거 나도 써봐야겠다"는 첫 시도 1개를 들고 나가기 | "월요일에 회사에서 시도해볼 것 1개" |

**중요**: 3시간 강의는 4주 과외의 압축본이 **아니다**. 두 트랙은 같은 분야를 다른 목적으로 다룬다:

- **3시간 강의** = 콘텐츠 전수 (감각·충격·첫 시도)
- **4주 과외** = 메타-스킬 전수 (학습 시스템 자체를 만들기)

3시간으로 메타-스킬을 심는 건 불가능하고, 4주로 단순한 콘텐츠 전수만 하면 끝나는 시점에 이미 노후화된다. 그래서 분리.

## 두 트랙이 공유하는 것

골격은 다르지만 **공유 자산**은 있다:

- **[필수 용어집](./glossary.md)** — AI 기초 / 프롬프트·컨텍스트 / 에이전트·하네스 / 워크플로우. 두 트랙 모두에서 청중·학습자 막힐 때 1초 컷용.
- **데모 자산** — 강의자 본인 프로젝트 1개. Claude Code 설치된 실제 작업물. 두 트랙 모두 라이브 데모의 핵심.
- **1차 출처 리스트** — Anthropic 공식 문서, Karpathy 영상, Simon Willison 블로그 등. 학습자에게 "여기서 봐서 정리했다"를 보여줄 때 쓰는 출처들.

## 차시 구조 (트랙별로 다름)

### 4주 과외 — 코칭형 4단
1. **이번 주 질문** (15분) — 학습자가 가져온 자기 업무 맥락 질문 1개
2. **함께 탐색** (60분) — 강의자가 사고 과정을 시연
3. **혼자 한 번 더** (30분) — 학습자가 즉석에서 같은 패턴으로
4. **다음 주까지 자기 사이클** (15분) — 1주 동안 본인 업무에 적용

각 차시 첫 30분은 별도로 지난 주 회고가 들어간다. 진짜 신규 콘텐츠는 90분.

### 3시간 강의 — 감각 전수형 3단
1. **개념 (10분)** — 핵심 1~2개만 짧게
2. **라이브 데모 (30분)** — 실제 작동을 보여줌
3. **청중 액션 + Q&A (10분)** — 따라하기 또는 액션 플래닝

이게 매 교시 동일.

## 강의자 노트

### 4주 과외 운영
- **답을 주지 말 것**. "그 자료를 어디서 찾을 것 같아요?"만 물을 것.
- **"나도 모른다"를 자주 보여줄 것**. 학습자가 "모르는 게 정상이고 찾아가는 거구나"를 체득해야 함.
- **회고가 콘텐츠의 절반**. 매 차시 첫 30분 = 지난 주 사이클 회고.

### 3시간 강의 운영
- **데모가 강의의 90%**. 슬라이드는 거들 뿐.
- **본인 프로젝트로 라이브** — 가짜 예제로는 충격이 안 옴.
- **백업 영상 필수** — 인터넷·계정·API 다 끊겨도 보여줄 수 있게.

### 사전 준비
- **강의자**: 노트북 + Claude Code 설치된 실제 프로젝트 1개 + 백업 데모 영상.
- **수강자/청중**:
  - 4주 과외: Claude Pro/Max 결제, Claude Code 설치, 자기 업무 영역 1개 결정
  - 3시간 강의: 노트북 + Claude.ai 계정만
