# Track: 글쓰기

Lv 1 트랙 중 하나. 바탕화면에 완성된 글 한 편을 남긴다.

## 1. Collect

AskUserQuestion 한 번에:

- 장르 — A) 에세이 B) 블로그 글 C) 소설 한 장면 D) 자유
- 주제 또는 첫 아이디어 (없으면 "제안해주세요" 허용, Claude가 5개 제안)
- 분량 — A) 짧게(500자) B) 보통(1500자) C) 길게(3000자+)
- 톤 — A) 가볍고 유머러스 B) 진지·감성적 C) 정보 전달형

주제가 "제안해주세요"면 Claude가 분위기에 맞는 후보 5개를 AskUserQuestion으로 되묻는다.

## 2. Prepare

```bash
mkdir -p ~/Desktop/my-writing
```

파일명은 주제 기반 slug. 예: `~/Desktop/my-writing/외로운-로봇.md`.

## 3. Create

Write로 초안 작성. 원칙:

- 첫 문장이 독자를 끌어당긴다
- "~할 수 있습니다", "~에 대해 알아보겠습니다" 같은 AI 티 나는 표현 금지
- 문장은 짧게. 한 단락 3~4줄 이내
- 구체적인 묘사·감각적 표현

장르별:
- **에세이** — 개인 경험에서 출발, 보편적 통찰로 확장
- **블로그** — 소제목으로 구조화, 검색자 의도 고려
- **소설** — 장면 중심, 대화체, 인물 감정 구체화
- **자유** — 사용자가 요청한 대로

## 4. Verify

```
{파일경로} 에 저장했어요.

읽어보시고 수정 요청 주세요.
예: "2번째 단락 더 감성적으로", "결말 바꿔줘", "전체 더 짧게"
```

피드백 루프: 해당 부분만 Edit로 수정 후 다시 보여준다.

## 5. Extend (선택)

AskUserQuestion: "이 글을 다른 형태로 바꿔볼까요?"

- A) 카드뉴스용 6컷 요약
- B) SNS용 짧은 버전 (280자)
- C) 뉴스레터 형식 (제목·소제목·CTA)
- D) 그만 (파일만 유지)

선택에 따라 같은 폴더에 새 파일로 저장 (예: `외로운-로봇-카드뉴스.md`).
