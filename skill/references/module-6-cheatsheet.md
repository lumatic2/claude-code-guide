## Mission 1 (m1): 명령어 속사 퀴즈

### 진행 방법

10문제를 AskUserQuestion으로 하나씩 출제한다.
각 문제 후 즉시 정답 여부를 보여준다. 맞은 개수를 추적한다.

### 문제

**Q1**: "대화 컨텍스트가 너무 길어졌을 때"
- 선택지: `/clear`, `/compact`, `/cost`, `/model`
- 정답: `/compact`
- 설명: `/compact`은 대화를 요약해서 토큰을 줄입니다. `/clear`는 완전 초기화.

**Q2**: "현재 대화를 완전히 초기화하려면"
- 선택지: `/compact`, `/clear`, `/exit`, `/help`
- 정답: `/clear`
- 설명: `/clear`는 모든 컨텍스트를 지우고 새로 시작합니다.

**Q3**: "실행 중인 작업을 중단하려면"
- 선택지: `Esc`, `Ctrl+C`, `Ctrl+Z`, `Ctrl+D`
- 정답: `Ctrl+C`
- 설명: `Ctrl+C`는 현재 실행 중인 작업을 중단합니다.

**Q4**: "특정 파일을 컨텍스트에 포함하려면"
- 선택지: `!파일경로`, `@파일경로`, `#파일경로`, `/파일경로`
- 정답: `@파일경로`
- 설명: `@`로 파일을 참조하면 Claude가 전체 내용을 읽습니다.

**Q5**: "셸 명령어를 직접 실행하려면"
- 선택지: `@명령어`, `!명령어`, `/명령어`, `$명령어`
- 정답: `!명령어`
- 설명: `!`로 시작하면 셸에서 직접 실행됩니다. 예: `!npm test`

**Q6**: "권한 모드를 전환하려면"
- 선택지: `Ctrl+Tab`, `Shift+Tab`, `Alt+Tab`, `Tab`
- 정답: `Shift+Tab`
- 설명: `Shift+Tab`으로 승인 모드 ↔ 전체 허용을 토글합니다.

**Q7**: "이전 상태로 롤백하려면"
- 선택지: `Ctrl+Z`, `Esc × 2`, `/undo`, `/revert`
- 정답: `Esc × 2`
- 설명: Esc를 두 번 빠르게 누르면 마지막 체크포인트로 롤백합니다.

**Q8**: "토큰 사용량을 확인하려면"
- 선택지: `/tokens`, `/cost`, `/usage`, `/stats`
- 정답: `/cost`
- 설명: `/cost`로 현재 세션의 토큰 사용량과 예상 비용을 확인합니다.

**Q9**: "빠른 응답 모드를 켜려면"
- 선택지: `/speed`, `/fast`, `/turbo`, `/quick`
- 정답: `/fast`
- 설명: `/fast`는 같은 모델에서 2.5배 빠른 응답을 제공합니다.

**Q10**: "복잡한 작업을 계획 모드로 설계하려면"
- 선택지: `/design`, `/plan`, `/architect`, `/think`
- 정답: `/plan`
- 설명: `/plan`으로 플랜 모드에 진입하면 코드 수정 없이 설계에 집중합니다.

### 결과 표시

총 점수를 보여준다:

- 10/10: "완벽합니다! Claude Code 마스터 🎓"
- 8-9/10: "훌륭해요! 거의 다 외웠습니다."
- 6-7/10: "좋아요! 몇 가지만 더 익히면 됩니다."
- 5 이하: "기본기를 다시 한번 복습하면 좋겠어요."

점수를 progress.json의 `quiz_scores`에 저장한다:
```json
{
  "quiz_scores": {
    "module_6_rapid": {
      "score": 8,
      "total": 10,
      "wrong": ["Q6", "Q8"]
    }
  }
}
```

## Mission 2 (m2): 나만의 치트시트 생성

### 진행 방법

progress.json의 `quiz_scores`를 읽어서 개인화된 치트시트를 생성한다.

### 치트시트 구조

Write 도구로 `~/.claude-study/my-cheatsheet.md`에 저장한다:

**틀린 문제가 있는 경우**:
```markdown
# 나의 Claude Code 치트시트

> /claude-study 학습 결과 기반으로 생성됨

## 🔴 복습 필요

| 명령어/단축키 | 설명 | 사용 시점 |
|--------------|------|-----------|
| Shift+Tab | 권한 모드 전환 | 도구 승인 방식을 바꾸고 싶을 때 |
| /cost | 토큰 사용량 확인 | API 비용을 체크하고 싶을 때 |

## 🟢 잘 알고 있어요

| 명령어/단축키 | 설명 |
|--------------|------|
| /compact | 대화 압축 (토큰 절약) |
| /clear | 컨텍스트 완전 초기화 |
| Ctrl+C | 실행 중단 |
| @파일경로 | 파일 컨텍스트 포함 |
| !명령어 | 셸 명령어 실행 |
| Esc × 2 | 체크포인트 롤백 |
| /fast | 빠른 응답 모드 |
| /plan | 플랜 모드 |

## 💡 추가 팁

- 큰 변경 전에는 항상 `!git commit -m "checkpoint"`
- 프롬프트에는 목표 + 파일 + 범위 + 완료 기준을 포함
- 에러 메시지를 프롬프트에 복사하면 디버깅 정확도 상승
```

**전부 맞은 경우**: "복습 필요" 섹션 없이 전체 레퍼런스 치트시트를 생성한다.
추가로 "🏆 만점! 보너스 팁" 섹션에 고급 팁을 추가한다:
- Sub-agents로 병렬 작업
- CLAUDE.md에 프로젝트 규칙 정의
- Hooks로 자동화 파이프라인 구축
